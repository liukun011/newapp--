# 小狸报告用户操作流程与现有文案记录

> 本文档基于当前小狸报告 App 代码梳理，用于交给另一位 AI 进行 UE 交互优化与文案优化。  
> 记录范围包含主要用户操作流程、页面细节、弹窗行为、状态反馈、现有文案和待优化点。  
> 已知交互问题/用户痛点栏可继续由产品或测试同学补充。

---

## 1. 产品基础信息

| 项目 | 当前内容 |
|---|---|
| 产品名称 | 小狸报告 |
| 产品类型 | 移动端 AI 访谈助手 / 尽调报告生成工具 |
| 目标用户 | 线下访谈、调研、尽调、业务分析人员 |
| 核心能力 | 访谈录音、实时/离线转写、资料上传、AI 问题洞察、AI 报告生成、模板管理、组织管理 |
| 主要端类型 | H5 / App WebView |
| 主要技术表现 | React + Vite + react-vant + Native Bridge |
| 核心主链路 | 登录 → 首页访谈记录 → 新建尽调 → 尽调详情 → 开始访谈 → 录音转写 → 上传资料 → 生成报告 |

## 2. 全局通用交互

### 2.1 页面结构与导航

| 项目 | 当前设计 |
|---|---|
| 页面容器 | 多数页面为移动端全屏布局，背景多为 `#F7F8FA` 或白色 |
| 顶部导航 | 二级页面通常左侧返回箭头，中间页面标题 |
| 原生返回 | `App.tsx` 统一监听 Native Back；部分页面通过 `requestNativeBack` 拦截并自定义返回 |
| 页面切换 | `framer-motion` 做前进/后退动画 |
| 状态持久化 | 当前页面、当前尽调、历史详情、模板预览等存入 `sessionStorage` |
| 全局录音状态 | Zustand store 维护当前尽调、访谈实例、标题、录音秒数、转写列表 |
| 录音悬浮窗 | 存在全局录音状态时展示，用于提示当前有访谈进行中 |

### 2.2 全局加载

| 场景 | 当前表现 | 当前文案 |
|---|---|---|
| 登录页首次加载 | 全屏白底转圈 | `资源加载中...` |
| 列表加载 | 页面中部 spinner | `加载中...` |
| 下拉刷新成功 | PullRefresh 成功态 | `加载成功` |
| 接口提交 | Toast.loading，常禁止点击 | `提交中...` / `保存中...` / `正在加载...` |
| 上传 | Toast.loading 或进度条 | `上传中...` / `上传中 {percent}%` |
| 报告生成 | 按钮禁用 + spinner | `报告生成中...` |

### 2.3 全局错误与 Toast

| 场景 | 当前文案 |
|---|---|
| 网络/接口异常 | `操作失败` / `网络异常，结束失败` / `加载预览失败` |
| 登录失败 | `登录失败` |
| 上传失败 | `上传失败` / `文件上传失败` |
| 保存失败 | `保存失败` |
| 删除失败 | `删除失败` |
| 权限不足 | `暂无管理权限` |
| 未找到业务对象 | `未找到尽调实例` / `尽调信息不存在` |
| 录音异常 | `录音开启失败，请点击重试` / Native 错误消息原样展示 |

### 2.4 全局弹窗规则

| 弹窗类型 | 当前表现 |
|---|---|
| react-vant Dialog.confirm | 居中确认弹窗，默认支持取消/确认 |
| 自定义删除弹窗 | 居中遮罩，白色圆角面板，点击遮罩关闭 |
| Popup 组织切换 | 居中弹窗，宽约 85%，最大宽 320px，顶部约 25% |
| 自定义重命名弹窗 | 居中遮罩，宽约 85%，最大宽 small，点击遮罩关闭 |
| VoiceInputModal | 底部/居中录音补充信息弹窗，含文本域和录音按钮 |

### 2.5 全局已知交互问题或用户痛点

| 编号 | 问题描述 | 影响程度 | 备注 |
|---|---|---|---|
| G-001 | 错误文案大量使用“失败/错误”，偏技术化 | 中 | 可统一为可恢复、可理解表达 |
| G-002 | 长任务如报告生成、资料解析的预估时间和后台继续提示不足 | 高 | 影响用户掌控感 |
| G-003 | Toast 文案风格不统一，有“您”和“你”混用 | 中 | 建议统一口吻 |
| G-004 | 多个危险操作确认按钮文案只写“确认”，后果表达不足 | 高 | 删除、覆盖、归档需要强化后果 |

## 7. 给优化 AI 的任务要求

请基于本文档记录的当前小狸报告流程、交互细节与现有文案，输出 UE 交互优化和文案优化方案。

### 7.1 输出 UE 问题清单

- 按流程逐项分析：登录、首页、新建尽调、详情、录音、资料、报告、问题清单、历史访谈、模板、组织管理
- 每个问题说明：当前表现、用户影响、建议方案、优先级
- 优先处理 P0：影响主链路完成、录音安全感、报告生成、删除/覆盖风险的问题

### 7.2 输出文案优化表格

每条文案至少提供 2-3 个备选版本，并标注语气：

| 页面 | 场景 | 当前文案 | 优化版本 A | 优化版本 B | 推荐版本 | 说明 |
|---|---|---|---|---|---|---|

文案要求：

- 统一使用“你”，除法律协议外不使用“您”
- 少用“失败/错误”，优先使用“暂时无法完成”“遇到点小问题”“请稍后重试”等可恢复表达
- Toast 尽量不超过 15 字
- 弹窗正文不超过两行，必要时用辅助说明承载风险
- 危险操作按钮必须表达后果，如“确认删除”“结束访谈并保存”“重新生成并覆盖”

### 7.3 输出弹窗与状态反馈优化建议

- 明确哪些操作需要二次确认：删除项目、删除资料、结束访谈、重新生成报告、归档、退出登录等
- 明确哪些操作应该用 Toast、页面状态、弹窗、进度条或顶部横幅
- 对长任务给出预计时间、进度、后台继续和完成提醒
- 对可恢复异常给出“原因 + 下一步动作”

### 7.4 输出空状态优化建议

每个空状态需包含：

- 当前为什么为空
- 用户下一步可以做什么
- 一个明确 CTA
- 必要时给出低焦虑解释

重点优化：

- 首页无尽调
- 资料为空
- 问题清单为空
- 历史访谈为空
- 搜索无结果
- 组织无权限
- 消息为空

### 7.5 输出优先级 Roadmap

- P0：新建/开始访谈、录音结束保存、资料解析、报告生成、删除/覆盖确认
- P1：空状态、组织切换、模板上传、权限说明、搜索无结果
- P2：语气统一、按钮微文案、成功反馈、动效和情绪化提示

请输出以下内容：

3. 弹窗优化建议
4. 空状态优化建议
