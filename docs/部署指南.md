# 部署配置指南

本文档说明如何将应用部署到不同环境。

## 部署流程

### 1. 构建应用

根据目标环境选择对应的构建命令：

```bash
# 开发环境
yarn build

# 测试环境
yarn build:test

# 生产环境
yarn build:prod
```

构建完成后，优化后的静态文件会输出到 `dist` 目录。

### 2. 验证构建产物

```bash
yarn preview
```

在浏览器中访问 `http://localhost:4173` 预览构建后的应用。

### 3. 部署到服务器

将 `dist` 目录的内容上传到你的 Web 服务器。

## 常见部署平台

### Nginx

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    root /var/www/talk-assistant/dist;
    index index.html;
    
    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
}
```

### Docker

创建 `Dockerfile`:

```dockerfile
# 构建阶段
FROM node:18-alpine as builder

WORKDIR /app

# 复制依赖文件
COPY package.json yarn.lock ./

# 安装依赖
RUN yarn install --frozen-lockfile

# 复制源代码
COPY . .

# 构建应用 (根据需要修改构建命令)
ARG BUILD_ENV=production
RUN yarn build:${BUILD_ENV}

# 生产阶段
FROM nginx:alpine

# 复制构建产物
COPY --from=builder /app/dist /usr/share/nginx/html

# 复制 Nginx 配置
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

构建不同环境的 Docker 镜像：

```bash
# 开发环境
docker build --build-arg BUILD_ENV=development -t talk-assistant:dev .

# 测试环境
docker build --build-arg BUILD_ENV=test -t talk-assistant:test .

# 生产环境
docker build --build-arg BUILD_ENV=prod -t talk-assistant:prod .
```

### Vercel

1. 安装 Vercel CLI:
   ```bash
   npm i -g vercel
   ```

2. 部署：
   ```bash
   # 开发环境
   vercel --build-env VITE_API_BASE_URL=http://dev.example.com/api
   
   # 测试环境
   vercel --build-env VITE_API_BASE_URL=http://test.example.com/api
   
   # 生产环境
   vercel --prod --build-env VITE_API_BASE_URL=https://api.example.com
   ```

或在 Vercel Dashboard 中配置环境变量。

### Netlify

1. 在 `netlify.toml` 中配置：

```toml
[build]
  command = "yarn build:prod"
  publish = "dist"

# 开发环境
[context.development.environment]
  VITE_API_BASE_URL = "http://dev.example.com/api"

# 测试环境
[context.staging.environment]
  VITE_API_BASE_URL = "http://test.example.com/api"

# 生产环境
[context.production.environment]
  VITE_API_BASE_URL = "https://api.example.com"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

2. 部署：
   ```bash
   netlify deploy --prod
   ```

### 阿里云 OSS / 腾讯云 COS

1. 构建应用：
   ```bash
   yarn build:prod
   ```

2. 上传到对象存储：
   ```bash
   # 使用 ossutil (阿里云)
   ossutil cp -r dist/ oss://your-bucket/
   
   # 使用 coscmd (腾讯云)
   coscmd upload -r dist/ /
   ```

3. 配置 CDN 和自定义域名

## CI/CD 配置示例

### GitHub Actions

创建 `.github/workflows/deploy.yml`:

```yaml
name: Deploy

on:
  push:
    branches:
      - main        # 生产环境
      - develop     # 开发环境
      - test        # 测试环境

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'yarn'
      
      - name: Install dependencies
        run: yarn install --frozen-lockfile
      
      - name: Build (Production)
        if: github.ref == 'refs/heads/main'
        run: yarn build:prod
      
      - name: Build (Test)
        if: github.ref == 'refs/heads/test'
        run: yarn build:test
      
      - name: Build (Development)
        if: github.ref == 'refs/heads/develop'
        run: yarn build
      
      - name: Deploy to Server
        run: |
          # 这里添加你的部署脚本
          # 例如: rsync, scp, 或使用云服务商的 CLI
```

### GitLab CI

创建 `.gitlab-ci.yml`:

```yaml
stages:
  - build
  - deploy

variables:
  NODE_VERSION: "18"

# 开发环境
build:dev:
  stage: build
  image: node:${NODE_VERSION}
  only:
    - develop
  script:
    - yarn install --frozen-lockfile
    - yarn build
  artifacts:
    paths:
      - dist/
    expire_in: 1 day

# 测试环境
build:test:
  stage: build
  image: node:${NODE_VERSION}
  only:
    - test
  script:
    - yarn install --frozen-lockfile
    - yarn build:test
  artifacts:
    paths:
      - dist/
    expire_in: 1 day

# 生产环境
build:prod:
  stage: build
  image: node:${NODE_VERSION}
  only:
    - main
  script:
    - yarn install --frozen-lockfile
    - yarn build:prod
  artifacts:
    paths:
      - dist/
    expire_in: 1 week

deploy:prod:
  stage: deploy
  only:
    - main
  script:
    - echo "Deploy to production server"
    # 添加部署脚本
```

## 环境变量安全

### ⚠️ 重要提醒

1. **前端环境变量是公开的**
   - 所有 `VITE_` 前缀的变量都会被打包到前端代码中
   - 可以在浏览器的开发者工具中查看
   - **不要存储敏感信息**（API 密钥、密码等）

2. **敏感信息应该在后端管理**
   - API 密钥应该存储在后端服务器
   - 前端只存储 API 的基础地址
   - 使用后端代理转发敏感请求

3. **使用环境变量平台功能**
   - Vercel、Netlify 等平台提供安全的环境变量管理
   - 在平台 Dashboard 中配置，而不是提交到代码仓库

## 故障排查

### 构建失败
- 检查 Node.js 版本是否匹配 (推荐 16+)
- 清除缓存: `rm -rf node_modules yarn.lock && yarn install`
- 检查环境变量是否正确配置

### 部署后白屏
- 检查浏览器控制台错误信息
- 确认静态资源路径是否正确
- 检查服务器是否正确配置 SPA 路由

### API 请求失败
- 确认 API 地址配置正确
- 检查 CORS 配置
- 验证网络连接和防火墙设置

## 监控和维护

- **错误追踪**: Sentry, Bugsnag
- **性能监控**: Google Analytics, Vercel Analytics
