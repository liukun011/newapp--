# 环境配置指南

## 环境说明

本项目使用 Vite 作为构建工具，支持三种不同的运行环境：

### 1. 开发环境 (Development)
- **API 地址:** `http://dev.example.com/api`
- **用途:** 本地开发和调试
- **配置文件:** `.env.development`

### 2. 测试环境 (Test)
- **API 地址:** `http://test.example.com/api`
- **用途:** 功能测试和集成测试
- **配置文件:** `.env.test`

### 3. 生产环境 (Production)
- **API 地址:** `https://api.example.com`
- **用途:** 正式线上环境
- **配置文件:** `.env.production`

## 环境变量说明

### VITE_API_BASE_URL
- **类型:** String
- **必填:** 是
- **说明:** API 接口的基础地址
- **示例:**
  ```
  VITE_API_BASE_URL=http://dev.example.com/api
  ```

### VITE_ENV
- **类型:** String
- **必填:** 否
- **说明:** 环境标识符
- **可选值:** `development` | `test` | `production`

## 使用方法

### 本地开发

#### 启动开发环境
```bash
yarn dev
# 或
yarn dev --mode development
```

#### 启动测试环境
```bash
yarn dev:test
# 或
yarn dev --mode test
```

### 构建部署

#### 构建开发环境
```bash
yarn build
# 或
yarn build --mode development
```
构建产物会使用开发环境的 API 地址。

#### 构建测试环境
```bash
yarn build:test
# 或
yarn build --mode test
```
构建产物会使用测试环境的 API 地址。

#### 构建生产环境
```bash
yarn build:prod
# 或
yarn build --mode production
```
构建产物会使用生产环境的 API 地址。

## 环境配置文件优先级

Vite 会按以下优先级加载环境变量文件：

1. `.env.[mode].local` (优先级最高，仅本地使用，不提交到版本控制)
2. `.env.[mode]` (环境特定配置)
3. `.env.local` (所有环境的本地覆盖，不提交到版本控制)
4. `.env` (所有环境的默认配置)

**示例:**
- 运行 `yarn dev` 时，会加载 `.env.development`
- 运行 `yarn build:test` 时，会加载 `.env.test`
- 运行 `yarn build:prod` 时，会加载 `.env.production`

## 在代码中使用环境变量

### 通过 config.ts
推荐使用统一的配置文件访问环境变量：

```typescript
import config from '@/config';

// 获取 API 基础地址
console.log(config.apiBaseUrl);

// 判断当前环境
if (config.isDev) {
  console.log('开发环境');
}

if (config.isTest) {
  console.log('测试环境');
}

if (config.isProd) {
  console.log('生产环境');
}
```

### 直接访问环境变量
也可以直接使用 Vite 提供的 `import.meta.env`：

```typescript
// 获取环境变量
const apiUrl = import.meta.env.VITE_API_BASE_URL;
const mode = import.meta.env.MODE;

// 判断是否为生产环境
const isProd = import.meta.env.PROD;
const isDev = import.meta.env.DEV;
```

## 环境变量命名规范

在 Vite 中，只有以 `VITE_` 开头的环境变量才会被暴露到客户端代码中。

✅ **正确示例:**
```
VITE_API_BASE_URL=http://dev.example.com/api
VITE_APP_TITLE=Talk Assistant
VITE_ENABLE_MOCK=true
```

❌ **错误示例:**
```
API_BASE_URL=http://dev.example.com/api  # 不会被暴露
APP_TITLE=Talk Assistant                 # 不会被暴露
```

## 本地开发自定义配置

如果你需要在本地使用不同的配置（例如连接本地后端），可以创建 `.env.local` 或 `.env.development.local` 文件：

```bash
# .env.development.local
VITE_API_BASE_URL=http://localhost:3000/api
```

**注意:** `.env.*.local` 文件已在 `.gitignore` 中被忽略，不会提交到版本控制系统。

## 验证环境配置

启动应用后，打开浏览器控制台，你会看到当前的环境配置信息（仅在非生产环境下显示）：

```
🌟 当前环境配置: {
  环境: 'development',
  API地址: 'http://dev.example.com/api'
}
```

## 常见问题

### Q1: 环境变量修改后没有生效？
**A:** 需要重启 dev server。Vite 的环境变量在构建时就已确定，运行时无法动态修改。

### Q2: 如何在生产环境隐藏调试信息？
**A:** 使用 `config.isProd` 进行条件判断：
```typescript
if (!config.isProd) {
  console.log('调试信息');
}
```

### Q3: 如何添加新的环境？
**A:** 
1. 创建新的环境变量文件，如 `.env.staging`
2. 在 `package.json` 中添加对应的脚本：
   ```json
   "dev:staging": "vite --mode staging",
   "build:staging": "tsc && vite build --mode staging"
   ```
3. 在 `config.ts` 中添加环境判断逻辑

## 安全注意事项

⚠️ **重要提醒:**

1. **不要在环境变量中存储敏感信息**（如密钥、密码等），因为这些变量会被打包到前端代码中，任何人都可以在浏览器中查看。

2. **API 密钥应该在后端管理**，前端只存储 API 的基础地址。

3. **不要将 `.env.local` 和 `.env.*.local` 文件提交到版本控制系统**。

4. **团队协作时**，应该将 `.env.example` 文件提交到版本控制，作为环境变量的模板。
